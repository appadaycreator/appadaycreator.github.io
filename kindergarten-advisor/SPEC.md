## v2.0 - M6 目標設定・達成度トラッキング機能追加（2026-06-10）

### 新機能: M6 目標設定・達成度トラッキング

#### 概要
診断完了後に「次回までの目標」を設定でき、再診断時に前回の目標に対する達成度をチェック・追跡できる機能。リピート利用を促進し、ユーザーの継続利用率を向上させる。

#### 実装内容

**1. 目標設定UI（モーダル）**
- 診断結果表示後に「🎯 次回の目標を設定する」ボタン（紫色）を追加
- クリックでモーダルダイアログを表示
- 目標テキスト入力フォーム（100文字まで）
- 文字数カウンター表示（X/100）
- 優先度選択（高/中/低）
- 保存 / スキップボタン

**2. データ保存（localStorage）**
- 目標情報を localStorage に保存
  - `kindergarten_histories[].goal`: { text, priority, timestamp }
- 達成度記録を別に保存
  - `kindergarten_goal_achievements`: [{ goalText, status, timestamp }]

**3. 前回目標の表示・達成度チェック**
- 再診断時、診断結果画面に「🎯 前回までの目標達成状況」セクションを表示
- 前回の目標テキストを表示（優先度ラベル付き）
- 達成度選択ボタン（3択）：
  - ✅ 達成できた
  - ⚠️ 部分的に達成
  - ❌ 達成できなかった
- タップ時に localStorage に記録

**4. 達成率統計表示**
- 過去の達成履歴を集計して表示
  - 達成数 / 部分的達成数 / 未達成数
  - 達成率（%）を計算・表示
- 例：「達成2回 / 部分的1回 / 未達成0回 / 達成率: 67%」

#### UI/UX 改善（P2）

- **入力フォーム**
  - placeholder: "例: 共働きの場合の保活スケジュールを調べる..."
  - フォーカス時のビジュアルフィードバック（既存のボーダーカラー変更）
  
- **バリデーション**
  - 空文字チェック（alert で警告）
  
- **文字数カウンター**
  - リアルタイム表示（X/100）

#### トークン使用量削減
- 1ファイル Read 1回のみ：index.html を1回だけ読む
- Edit 複数回で実装：CSS → HTMLマークアップ → JavaScript関数
