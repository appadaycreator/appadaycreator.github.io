## 概要
食事の写真を登録・管理するウェブアプリです。GitHub Pages で公開し、Supabase を用いてデータを保存します。バニラJavaScriptで構築され、ビルド不要で静的ホスティングに対応しています。

## 主な画面
- **index.html**: 食事登録および一覧表示（メイン画面）
- **howto.html**: 簡易的な使い方説明
- **usage.html**: 詳細な使い方説明
- **categories.html**: カテゴリ編集
- **ingredients.html**: 食材編集
- **login.html**: ログイン画面
- **terms.html**: 利用規約
- **disclaimer.html**: 免責事項
- **privacy.html**: プライバシーポリシー
- **contact.html**: 問い合わせ
- **404.html**: 存在しないURLにアクセスした際に表示

## UI/デザイン仕様

### デザインシステム
- **カラーパレット**: インディゴ系（#6366f1）をプライマリカラーとして使用
- **グラデーション**: プライマリ、セカンダリ、サクセスカラーにグラデーション効果を適用

- **トランジション**: cubic-bezier(0.4, 0, 0.2, 1)を使用

## データ保存

### ローカルストレージ
- カテゴリ一覧 (`categories`)
- 食材一覧 (`ingredients`)
- デバッグモード設定 (`debug_mode`)
- テーマ設定 (`theme`)

### Supabase データベース
- **meals テーブル**:
  - `_id` (UUID, Primary Key)
  - `name` (TEXT, NOT NULL)
  - `capturedate` (DATE, DEFAULT CURRENT_DATE)
  - `user_id` (UUID, Foreign Key to auth.users)
  - `category` (TEXT)
  - `ingredients` (TEXT[])
  - `image` (TEXT)
  - `is_favorite` (BOOLEAN, DEFAULT FALSE) - お気に入り機能
  - `last_suggested_date` (DATE) - 献立候補機能用
  - `suggestion_count` (INTEGER, DEFAULT 0) - 献立候補機能用
  - **栄養データ（統計・グラフ機能用）**:
    - `calories` (INTEGER) - カロリー（kcal）
    - `protein` (DECIMAL(5,2)) - タンパク質（g）
    - `carbs` (DECIMAL(5,2)) - 炭水化物（g）
    - `fat` (DECIMAL(5,2)) - 脂質（g）
    - `fiber` (DECIMAL(5,2)) - 食物繊維（g）
    - `sodium` (DECIMAL(5,2)) - ナトリウム（mg）
  - `created_at` (TIMESTAMP WITH TIME ZONE)
  - `updated_at` (TIMESTAMP WITH TIME ZONE)

### Row Level Security (RLS) - 最適化済み
```sql
-- ユーザーが自分の料理のみアクセス可能（パフォーマンス最適化済み）
CREATE POLICY "Users can view own meals" ON meals
  FOR SELECT USING ((select auth.uid()) = user_id);

CREATE POLICY "Users can insert own meals" ON meals
  FOR INSERT WITH CHECK ((select auth.uid()) = user_id);

CREATE POLICY "Users can update own meals" ON meals
  FOR UPDATE USING ((select auth.uid()) = user_id);

CREATE POLICY "Users can delete own meals" ON meals
  FOR DELETE USING ((select auth.uid()) = user_id);
```
